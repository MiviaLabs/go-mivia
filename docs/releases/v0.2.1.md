# Mivia v0.2.1 Release Notes

Date: 2026-06-04

## Highlights

- Fixed MCP `project_id` handling across Work Plan, Work Task, automation, and workspace worktree tool paths so callers can use the canonical project id field consistently.
- Fixed automatic Work Plan status-trigger execution through the external automation runner: `active` Work Plans with enabled automatic automations now queue runs over ready tasks, the sidecar runner claims them, launches Codex CLI, and returns runs to verifier-required state.
- Added `projects.work_tasks.list` as a supported alias for `projects.work_tasks.list_open`.
- Expanded Work Task create metadata with `files_to_read`, `files_to_edit`, `review_gate`, create-time non-terminal `status`, `failure_criteria`, and `decomposition_quality`.
- Persisted and returned Work Task read/edit/review metadata through in-memory and Ladybug stores.
- Allowed safe project-relative paths in Work Task descriptions while continuing to reject roots, traversal, UNC paths, home-directory shortcuts, drive prefixes, secrets, raw prompts, raw source dumps, provider payloads, and PII.
- Improved dashboard automation-run visibility with run metadata, plan/task linkage, runner state, attempt count, queue timestamps, and safe summaries.

## Safety And Governance

- Automation remains an executor only. It does not complete Work Tasks, validate claims, score confidence, promote knowledge, or bypass independent review.
- Successful Codex CLI execution moves an automation run to `verifying` with verifier-required metadata. The workflow orchestrator still owns review refs, verifier refs, Evidence Graph outcomes, confidence scoring when reusable, Knowledge Promotion candidates, Work Task completion, and Work Plan completion.
- Work Task create may set only non-terminal statuses. Terminal states still require lifecycle tools and governance evidence.
- Work Task packet metadata stays bounded and metadata-only. Raw prompts, completions, source dumps, raw stderr, provider payloads, secrets, roots, external URLs, skipped sensitive content, and PII remain prohibited.
