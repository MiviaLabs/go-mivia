# Mivia v0.2.0 Release Notes

Date: 2026-06-04

## Highlights

- Added governed Workflow TOML import and compile flow for decomposition/planning, Work Plan implementation, and code-review bug planning.
- Added Work Plan and Work Task governance for low-intelligence worker decomposition, independent review, verifier refs, Evidence Graph metadata, confidence scoring, and Knowledge Promotion gates.
- Added metadata-only project automation definitions and automation runs over existing Work Plans and ready Work Tasks.
- Added `mivia-automation-runner` for external Codex CLI execution from the user's logged-in local environment.
- Added automatic workflow support that can queue external runs only after required review tasks are complete.
- Added Linux, macOS, and native WSL direct runner support, plus an explicit WSL-to-Windows Codex bridge through `--codex-launcher windows-cmd`.

## Safety And Governance

- Workflow TOML remains compile-only metadata; it does not execute directly.
- Automation cannot bypass Work Plans, Work Tasks, generated review tasks, independent review refs, orchestrator verifier refs, Evidence Graph outcomes, confidence scoring, or Knowledge Promotion gates.
- Automation records, Work Task records, and runner attempt records stay metadata-only and must not store raw prompts, source dumps, raw stderr, secrets, roots, provider payloads, external URLs, or PII.
- Automatic code-review bug planning creates bug Work Plans only for independently confirmed, evidence-backed bugs. It must not auto-implement speculative or unreviewed findings.
